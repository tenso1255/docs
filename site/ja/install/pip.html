TensorFlow 2.0 Alphaが利用可能です

  • tensorflow==2.0.0-alpha0 —Preview TF 2.0 Alpha CPUのみのビルド(不安定)
  • tensorflow-gpu==2.0.0-alpha0 —Preview TF 2.0 Alpha GPUサポート付きのビルド (不安定、UbuntuおよびWindows)

追加パッケージ

  • tensorflow —CPU用の最新の安定版リリース(UbuntuおよびWindows)
  • tensorflow-gpu —GPUサポート付きの最新の安定版リリース (UbuntuおよびWindows)
  • tf-nightly —CPU用のプレビューnightlyビルド (不安定)
  • tf-nightly-gpu —GPUサポート付きのプレビューnightlyビルド (不安定、UbuntuおよびWindows)

システム要件

  • Ubuntu 16.04 以降 (64-bit)
  • macOS 10.12.6 (Sierra) 以降 (64-bit) (GPUサポートは無し)
  • Windows 7 以降 (64-bit) (Python 3 のみ)
  • Raspbian 9.0 以降

ハードウェア要件

  • TensorFlow 1.6 以降、バイナリーはAVX手順を使用しており、古いCPU上で実行できないおそれがあります。
  • GPUサポートガイドを読み、UbuntuもしくはWindows上でCUDA®対応のGPUカードをセットアップしてください。

2. 仮想環境を作成する (推奨)

Python仮想環境は、システムからパッケージのインストールを分離するために使用されます。

Ubuntu / mac OS

Pythonインタープリターを選択、その情報を./venvディレクトリーに保持することで、 新しい仮想環境を作成します:

{% dynamic if request.query_string.lang == "python2" %} 
virtualenv --system-site-packages -p python2.7 ./venv
{% dynamic else %} 
virtualenv --system-site-packages -p python3 ./venv
{% dynamic endif %}

シェル固有のコマンドを使用して仮想環境を有効にします: 

source ./venv/bin/activate  # sh, bash, ksh, or zsh

virtualenvば有効な場合、シェルプロンプトの先頭は(venv)となっています。

ホストシステムの設定に影響を与えずに仮想環境内にパッケージをインストールします。 pipをアップグレードすることから始めます: 

pip install --upgrade pip

pip list  # show packages installed within the virtual environment

そして、後ほどvirtualenvを終了します: 

deactivate  # don't exit until you're done using TensorFlow

Windows

Pythonインタープリターを選択、その情報を.\venvディレクトリーに保持することで、 新しい仮想環境を作成します:

{% dynamic if request.query_string.lang == "python2" %}
TensorFlow is not supported on Windows with Python 2.7
{% dynamic else %} 
virtualenv --system-site-packages -p python3 ./venv
{% dynamic endif %}

仮想環境を有効にします: 

.\venv\Scripts\activate

ホストシステムの設定に影響を与えずに仮想環境内にパッケージをインストールします。 pipをアップグレードすることから始めます: 

pip install --upgrade pip

pip list  # show packages installed within the virtual environment

そして、後ほどvirtualenvを終了します: 

deactivate  # don't exit until you're done using TensorFlow

Conda

TensorFlow提供のpipパッケージを推奨しますが、 コミュニティサポートの Anacondaパッケージ が利用可能です。

Pythonインタープリターを選択、その情報を./venvディレクトリーに保持することで、 新しい仮想環境を作成します:

{% dynamic if request.query_string.lang == "python2" %} 
conda create -n venv pip python=2.7
{% dynamic else %} 
conda create -n venv pip python=3.6  # select python version
{% dynamic endif %}

仮想環境を有効にします:

source activate venv

仮想環境内で、完全なURLを使用してTensoFlowのpipパッケージをインストールしてください: 

pip install --ignore-installed --upgrade packageURL

そして、後ほどvirtualenvを終了します: 

source deactivate

3. TensorFlowのpipパッケージをインストールする

PyPIから、以下のTensorFlowパッケージの中から一つ選択してインストールしてください:

  • tensorflow —CPUのみの最新の安定版リリース (初心者におすすめ)
  • tensorflow-gpu —GPUサポート付きの最新の安定版リリース (UbuntuおよびWindows)
  • tf-nightly —CPUのみのプレビューnightlyビルド (不安定)
  • tf-nightly-gpu —GPUサポート付きのプレビューnightlyビルド (不安定、UbuntuおよびWindows)
  • tensorflow==2.0.0-alpha0 —CPUのみのプレビュー TF 2.0 Alpha ビルド (不安定)
  • tensorflow-gpu==2.0.0-alpha0 —GPUサポート付きのTF 2.0 Alpha ビルド (不安定、UbuntuおよびWindows)
パッケージ依存関係は自動的にインストールされます。これらは setup.py ファイルのREQUIRED_PACKAGESの下に一覧があります。

Virtualenvをインストールする

pip install --upgrade tensorflow

インストールされたことを確認します:

python -c "import tensorflow as tf; tf.enable_eager_execution(); print(tf.reduce_sum(tf.random_normal([1000, 1000])))"

システムインストール

{% dynamic if request.query_string.lang == "python2" %} 
pip install --user --upgrade tensorflow  # install in $HOME

インストールされたことを確認します:

python -c "import tensorflow as tf; tf.enable_eager_execution(); print(tf.reduce_sum(tf.random_normal([1000, 1000])))"
{% dynamic else %} 
pip3 install --user --upgrade tensorflow  # install in $HOME

インストールされたことを確認します:

python3 -c "import tensorflow as tf; tf.enable_eager_execution(); print(tf.reduce_sum(tf.random_normal([1000, 1000])))"
{% dynamic endif %}
成功: TensorFlowがインストールされました。チュートリアルを読んで始めましょう。
